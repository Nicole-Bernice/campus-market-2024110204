# Task5 校园轻集市项目规划
## 一、页面清单
1. CreateUser.vue 本地身份创建页
2. Home.vue 集市首页
3. MarketList.vue 信息列表+搜索筛选页
4. ItemDetail.vue 信息详情、收藏、砍价弹窗页面
5. Publish.vue 四类信息发布表单页
6. Message.vue 消息会话中心
7. Personal.vue 个人中心（我的发布、我的收藏）
8. Dashboard.vue 数据趋势看板

## 二、功能模块
1. 用户模块：本地档案创建、用户信息持久化、信用分展示
2. 集市信息模块：二手/失物/拼单/跑腿信息发布、列表筛选、详情查看、状态更新
3. 收藏模块：收藏/取消收藏、个人中心收藏列表同步
4. 消息互动模块：聊天会话、模拟砍价、自动回复、未读消息标记
5. 数据可视化模块：前端统计数据，ECharts图表展示类型/校区/状态分布

## 三、开发顺序
1. 环境准备：pnpm install安装依赖，启动前端dev服务与JSON Server Mock服务
2. 路由搭建：完善src/router全部页面路由、动态详情路由
3. 接口封装：src/api封装用户、商品、收藏、消息、公告CRUD接口
4. 状态管理：搭建Pinia仓库，管理用户、商品、收藏、消息全局状态
5. 公共组件开发：信息卡片、筛选栏、发布表单、聊天框、图表容器
6. 业务页面开发：身份创建 → 首页 → 列表 → 详情 → 发布 → 消息 → 个人中心 → 看板
7. 整体联调、bug修复、功能自测

## 四、开发重点
1. 严格遵循分层目录规范，代码不堆积在页面文件内
2. Axios统一接口封装，页面不直接写请求地址
3. Pinia保证多页面数据状态同步（收藏、消息、发布信息）
4. 复用组件抽离，减少重复代码，提升可维护性
5. Mock接口完整CRUD流程，熟悉前后端分离交互逻辑
