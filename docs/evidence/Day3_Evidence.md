# Day3 实训完成证据
## 一、Task1：搭建Mock数据集db.json
我自主规划整套模拟数据结构，决定设计六张贴合校园集市业务的数据表：users、items、favorites、conversations、messages、notices，为了避免前后端字段不匹配问题，我主动对照`src/utils`下`constants.ts`里的枚举常量规范，统一所有字段命名、帖子分类、状态、校区标识，保证Mock数据和前端业务规范完全对齐，完成后提交db.json数据源版本记录。

## 二、Task2：部署JSON-Server模拟后端接口
为了快速模拟后端接口，我选择全局安装json-server 0.17.4，自行配置`pnpm mock`启动脚本，监听3000端口；我设计双终端并行开发方案，一个终端运行`pnpm dev`启动前端5173页面，另一个终端运行Mock服务3000接口。完成部署后我逐个在浏览器访问六个REST接口自测，检验全部查询、分类筛选接口能否正常返回JSON数据，全程自主排查接口访问、数据返回相关问题，并留存自测截图。

## 三、Task3：分层封装Axios请求
我采用分层解耦的思路拆分接口代码，分为基础请求层和业务接口层：
1. `src/api/http.ts`：我创建统一axios实例，手动固定baseURL为`http://localhost:3000`，自主编写请求、响应拦截器，实现自动剥离接口外层data，后续页面调用接口不用重复写`.data`，简化页面代码；
2. `src/api/item.ts`：我复用http.ts封装好的实例，自行封装帖子全套业务接口，包含查询全部、按类型筛选、单条详情、新增帖子、修改帖子状态，将接口地址统一收拢，做到页面与接口地址解耦，方便后期维护修改。

## 四、Task4：商品列表页动态渲染数据
我独立改造`ListView.vue`页面，完整实现动态数据渲染功能：
1. 引入我封装好的接口函数，在页面挂载生命周期函数中自动加载全部帖子数据；
2. 我设计分类筛选功能，使用监听函数监听单选框筛选值，切换分类时自动发起对应筛选接口请求；
3. 考虑用户体验，我主动增加空状态提示文案并编写配套样式，当分类无匹配数据时友好展示提示文字；
4. 页面自测效果由我完整验证：全部分类均可正常渲染帖子卡片，切换筛选实时刷新内容，无数据时展示空提示，卡片查看详情按钮可正常携带id跳转详情页；
5. 自主踩坑修复：最初编写代码时接口取值写了`res.data`，页面空白无数据，我顺着拦截器逻辑排查问题，删除`.data`后列表正常渲染数据，记录下拦截器与返回值的对应逻辑。

## 五、实训踩坑与个人总结
本次实训中大部分方案、结构都是我自己思考设计，过程中遇到不少问题并独立解决：
1. Git提交漏文件：最初只提交ListView.vue页面，新增的EmptyState、ItemCard组件改动没有暂存，提交失败，之后我学会`git add .`批量暂存所有修改文件；
2. Axios实例复用问题：一开始我在item.ts重复新建axios实例，无法共享拦截器，后面改为导入统一http实例，统一共用拦截逻辑；
3. 接口返回值层级错误：不熟悉拦截器处理逻辑，多写一层data导致页面空白，通过打印返回数据排查根源；
4. 页面无数据排查思路：遇到列表只显示空状态时，我自主梳理排查流程，重启前后端双终端、直接访问3000接口校验数据源，逐步定位问题。
