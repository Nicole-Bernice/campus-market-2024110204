## 2. 注册登录设计说明
- 注册：前端收集用户名、密码表单，调用接口 POST /users 将用户数据写入 JSON Server 的 users 集合；
- 登录：根据输入用户名 GET /users?username=xxx 查询用户，匹配密码完成身份校验；
- 登录成功：调用状态库 setUser 方法存储用户对象，同时将用户信息序列化存入 localStorage；
- 页面初始化：App.vue onMounted 执行 recoverLogin，读取 localStorage 恢复登录状态；
- 退出登录：调用 logout，清空内存内用户信息，同时删除 localStorage 存储。

## 4. 状态持久化说明
1. 内存状态仅当前页面会话有效，浏览器刷新、页面关闭后状态直接丢失；
2. localStorage 属于浏览器本地持久化存储，数据长期保存在客户端，页面刷新后可读取；
3. 项目初始化时通过 recoverLogin 读取本地存储，自动恢复上次登录身份；
4. 本方案仅适用于前端实训Mock演示，未做密码加密，不满足线上生产安全标准。

## 10. 遇到的问题与解决方法
### 问题1：App.vue 报 Identifier 'useUserStore' has already been declared
- 原因：script 内重复导入状态库，重复实例化 userStore 变量；
- 解决：删除一组重复的 import 和变量声明，仅保留一次导入与实例化。

### 问题2：模板报错 v-else has no adjacent v-if
- 原因：单个 v-if 后并列两个 v-else，Vue语法不支持；
- 解决：使用 `<template v-else>` 包裹登录、注册两个RouterLink，统一归属一个else分支。

### 问题3：发布帖子 publisher 为 undefined
- 原因：模板与代码中错误使用 userStore.account，仓库内用户存储在 loginUser.username；
- 解决：统一修改为 userStore.loginUser.username，与状态仓库字段匹配。

### 问题4：ListView切换「失物招领」无数据
- 原因：单选框筛选标识与数据库type字段不匹配；
- 解决：修改单选框绑定值为lost，与数据type保持一致。

## 11. 今日反思
通过本次实训实现Mock注册登录与状态持久化，我理解了前端内存状态与本地持久化存储的区别：全局状态仅管理运行时页面状态，刷新页面即销毁，而localStorage可以长期缓存用户身份，提升用户体验。同时清晰区分了前端Mock认证与线上真实安全体系的边界，明白本地存储、明文密码仅适用于实训演示，生产环境必须搭配加密、JWT、后端鉴权等安全方案。另外在封装通用组件、统一页面交互反馈后，体会到组件化开发对代码复用、页面体验一致性的提升作用。
