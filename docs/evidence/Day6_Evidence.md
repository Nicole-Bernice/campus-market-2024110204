## 2. 注册登录设计说明
本项目Mock注册、登录完整流程如下：
- 注册时：前端收集用户名、密码、学院、年级等表单信息，通过axios POST请求写入JSON Server的`users`资源，db.json自动生成唯一用户id；
- 登录时：请求users接口获取全部用户数据，前端循环比对输入的用户名、密码，校验匹配则登录成功；
- 登录成功后：将当前完整用户对象保存到Pinia全局仓库，导航栏、发布页、个人中心可全局读取；
- 同时使用localStorage序列化保存当前用户基础信息，实现页面刷新后状态恢复；
- 退出登录时：清空Pinia仓库内loginUser数据，删除localStorage中用户缓存，自动跳转首页。

## 4. 状态持久化说明
### localStorage存在的必要性
1. Pinia仅为内存级状态管理，页面刷新、浏览器关闭后内存数据会直接丢失；
2. localStorage是浏览器本地持久化存储，可长期保存字符串格式用户数据，页面刷新后读取缓存自动恢复登录；
3. App项目初始化时执行`restoreLogin`方法，读取localStorage缓存回填Pinia仓库，无需重复登录；
4. 本项目仅为前端Mock实训登录方案，不具备线上生产环境安全加密、鉴权能力，仅用于模拟用户会话。

## 10. 遇到的问题与解决方法
### 问题1：点击导航栏「发布信息」页面空白，控制台报错`No match found for location with path "/publishing"`
- 故障原因：路由配置发布页面path为`/publish`，导航栏跳转路径为`/publishing`，两者路径不匹配，无对应路由规则；
- 解决方法：在`router/index.js`发布路由配置`alias: "/publishing"`路径别名，同时将`/publishing`加入路由守卫拦截名单；
- 修复效果：两条路径均可正常打开发布页面，路由404报错彻底消失。

### 问题2：控制台大量黄色警告`The next() callback in navigation guards is deprecated`
- 故障原因：Vue3废弃路由守卫`next(toPath)`写法，官方推荐直接return跳转路径；
- 解决方法：重写`router.beforeEach`全局守卫，删除`next、from`参数，通过return控制页面重定向；
- 修复效果：全部路由相关黄色警告清除，登录拦截业务逻辑保持不变。

### 问题3：页面`el-button`、`el-radio`等Element组件控制台提示无法解析
- 故障原因：未在项目入口`main.js`全局注册Element Plus，仅局部使用组件无法识别；
- 解决方法：在main.js顶部导入ElementPlus与配套css样式，执行`app.use(ElementPlus)`全局注册；
- 修复效果：所有Element UI表单、按钮、单选、卡片组件正常渲染，组件解析警告消除。

### 问题4：路由守卫顶部未全局导入Pinia仓库，抛出`useUserStore is not a function`，登录状态读取失效
- 故障原因：在router.beforeEach内部动态导入Pinia仓库，Pinia实例未初始化完成，调用时机错误；
- 解决方法：将`useUserStore`导入语句放置在`router/index.js`文件顶部全局导入，守卫内部直接实例化；
- 修复效果：Pinia仓库正常调用，登录状态全局读取正常，个人中心页面空白问题解决。
